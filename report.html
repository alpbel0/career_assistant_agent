1. Project Overview

The Career Assistant AI Agent is a self-evaluating, multi-agent system that automates professional communication with potential employers via Telegram. The system uses two LLMs in a pipeline: a Career Agent that generates responses and a Judge Agent that evaluates them against four EvalOps metrics. Only responses that score above the approval threshold are sent to the employer. Responses that fail are revised iteratively, while sensitive topics automatically route to human oversight.

The project covers the full LLMOps lifecycle: prompt engineering, retrieval-augmented generation (RAG) via ChromaDB, LLM-as-a-Judge evaluation, human-in-the-loop intervention routing, interaction logging, and a real-time monitoring dashboard.

4. EvalOps Evaluation Metrics

The Judge Agent (Gemini 2.0 Flash) evaluates every generated response on a 1–5 scale across four dimensions. Responses must achieve an overall score ≥ 4.0 to be sent automatically.

Truthfulness
Grounded in CV facts — no hallucination or invented details
Robustness
Resistant to prompt injection and off-topic manipulation
Helpfulness
Directly addresses the employer's question or request
Tone
Professional, clear, and polite language throughout

The judge also detects sensitive topics and sets trigger_human_intervention: true when needed, with a structured reason field.

5. Human Intervention Triggers

The system uses an LLM-based classifier (not keywords) to detect messages that require human judgment before responding. When triggered, the bot pauses and notifies the admin via Telegram with the full context and action buttons.

  • 💰 Salary Negotiation — Specific salary/equity numbers discussed
  • ⚖️ Legal Questions — Contracts, NDA, terms to sign
  • 🌫️ Ambiguous Offers — Formal offer needing acceptance/rejection
  • ❓ [UNCERTAIN] — Agent cannot answer from CV context

Admin receives action buttons: "Send Draft" or "Write Custom" (via /reply command which then shows a "Send" button for delivery).

8. Key Design Decisions

LLM-as-a-Judge over rule-based scoring

Instead of hardcoded scoring rules, a separate LLM (Gemini 2.0 Flash) evaluates each response. This allows nuanced, context-aware evaluation that captures tone and coherence — things keyword matching cannot.

LLM-based intervention classifier over keywords

Keyword detection for sensitive topics (salary, legal) fails with alternative phrasings. An LLM classifier handles paraphrases, indirect language, and non-English inputs correctly.

Hybrid CV context (static vs. RAG)

For short CVs (<2000 chars) the full text is injected directly into the prompt. For longer CVs, ChromaDB retrieves the most relevant chunks. This avoids context window waste while maintaining accuracy.

Revision loop with feedback propagation

When a response is rejected, the judge's feedback field is passed back to the career agent as explicit guidance. This creates a tight improvement loop rather than random retries.

9. Failure Cases & Lessons Learned

🔴 Failures Encountered

  • Bot hallucinated "14:00–17:00" availability — not in CV; fixed by tightening [UNCERTAIN] prompt rules
  • CV relevance pre-check blocked generic questions like "Tell me about yourself" — removed the check, delegated to Judge
  • LLM classifier flagged standard interview questions as off-topic — fixed with explicit negative examples in the prompt
  • Koyeb container had empty ChromaDB — CV was in .gitignore; fixed by committing cv.txt and re-indexing on startup

🟢 Lessons Learned

  • Prompt conflicts silently override each other — explicit, non-contradictory rules are critical
  • Conversation history can poison future responses — periodic inspection of history.json is necessary
  • LLM classifiers need explicit counter-examples, not just positive examples
  • Cloud deployments require CV data in the repo — stateful data cannot be assumed to persist
  • EvalOps is only as good as the judge prompt — judge quality directly affects the whole pipeline

10. Conclusion

The Career Assistant AI Agent demonstrates a functional EvalOps pipeline applied to a real-world use case. The system successfully automates professional employer communication, evaluates its own outputs using a second LLM, routes sensitive topics to human oversight, and logs every interaction for monitoring.

The project is deployed end-to-end: the bot and API run on Koyeb, the monitoring dashboard runs on GitHub Pages, and all data flows through a structured pipeline with no manual intervention required for standard career queries. Test cases confirm the system handles interview invitations, technical questions, and salary negotiations correctly.

Future improvements would include persistent storage for logs across deployments, a feedback loop from admin approvals back into the judge's calibration, and a more sophisticated conversation memory system.
